Custom Execution Documentation

Class Name: CustomExecution

Description: This class should provide the implementation for the custom scheduling policy.


public interface ExecutionInterface {

    void setReadyQueue(LinkedList readyQueue);

    void setWaitQueue(LinkedList waitQueue);

    void cpuExecution();

    LinkedList getProcessList();

    LinkedList[] getProcessOrder();

}

Module Details

  • This class should implement ExecutionInterface and override the above methods.


  • The method setReadyQueue sets the ready queue to be used for scheduling.


    Parameters:
    • Linked List of Process class(readyQueue)
    Return Type:
    • void

  • The method setWaitQueue sets the wait queue to be used for scheduling.


    Parameters:
    • Linked List of Process class(waitQueue)
    Return Type:
    • void

  • The method getProcessList returns a list of executed processes in the order of their execution.


    Parameters:
    • none
    Return Type:
    • Linked List of Process class(processList)

  • The method getProcessOrder returns an array of list of processes going to each cpu in the order of their exection. Each array has the list of processes which executes on that particular cpu.


    Parameters:
    • none
    Return Type:
    • Array of Linked List of Process Order class(processList)

  • The method cpuExecution contains the code according to the scheduling policy. That assign each cpu a process from the ready queue


    Parameters:
    • none
    Return Type:
    • none
    Details:
    • A loop is run for each core. For each index the scheduling policy selects a process from the ready queue and assigns it to that cpu.

    • It creates a thread of class CustomCPUExecution. It sets the process, ready queue and wait queue for that thread.

    • It calculates the start time and end time for each process.

    • According the order of execution of process, the processes are added to the process order list.

    • When execution of a process completes, it is then added into the process list.
